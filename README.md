# SimAnt iPad 版

iPad Safari で動かすための SFC エミュレーター。`files/` 配下のデスクトップ版を元に、タッチ操作・PWA 対応を加えたバージョンです。

## ファイル構成

```
ipad/
  index.html       # iPad 向け HTML (タッチパッド + レスポンシブレイアウト)
  manifest.json    # PWA マニフェスト
  icon.svg         # アイコン (アリのシルエット)
  src/
    cartridge.js   # エミュレーターコア (デスクトップ版と同一)
    bus.js
    cpu65816.js
    spc700.js
    ppu.js
    disasm.js
    emulator.js
```

## デプロイ手順 (3 つの選択肢)

### A. Mac の Python サーバーで配信 (お試し向け)

`files/ipad/` で:

```sh
python3 -m http.server 8080
```

iPad と同じ Wi-Fi に接続したうえで Safari で `http://<MacのIP>:8080/` を開く。
Mac の IP は「システム設定 → ネットワーク」で確認できます。

Mac を起動している間だけ動作。

### B. GitHub Pages にアップロード (推奨)

1. GitHub に空リポジトリを作成
2. `ipad/` フォルダの中身をプッシュ
3. リポジトリ設定 → Pages で「main ブランチ」を公開
4. 数分後、`https://<ユーザー名>.github.io/<リポジトリ名>/` で公開

iPad ではこの URL を Safari で開くだけ。Mac は不要。

### C. Netlify ドラッグ&ドロップ (一番手軽)

1. [Netlify Drop](https://app.netlify.com/drop) を開く
2. `ipad/` フォルダをブラウザにドラッグ
3. 即座に URL が払い出される

## iPad での操作方法

### ROM の読み込み

ROM ファイル (.sfc) を以下のいずれかの方法で iPad に転送:
- Mac から AirDrop
- iCloud Drive 経由
- メールで自分宛に送付

iPad の「ファイル」アプリに保存しておく。エミュレーターを開いたら **「📂 ROM」** ボタンをタップしてファイルを選択。

### コントローラ

3 つの入力方法に対応:

1. **画面のタッチパッド** (デフォルト)
   - 左下: 十字キー
   - 右下: A / B / X / Y
   - 上隅: L / R
   - 中央: SELECT / START

2. **Bluetooth 接続のキーボード**
   - Z=B、X=A、A=Y、S=X、Q=L、W=R、矢印=方向、Enter=Start、Shift=Select

3. **Bluetooth 接続のゲームパッド** (PS5 DualSense / Xbox 等)
   - iPadOS 14.5 以上
   - PS5 のフェイスボタン配置がそのまま SNES に対応

3 つは同時に有効。どれを使っても操作できます。

## 「ホーム画面に追加」してアプリ化する (推奨)

iPad ではこの操作で **ほぼネイティブアプリと同じ体験** にできます。

### 手順

1. Safari でこのページを開く
2. アドレスバー右の **共有ボタン** (□↑) をタップ
3. メニューを下にスクロールして **「ホーム画面に追加」** を選択
4. 名前を確認 (デフォルト: SimAnt) して右上の「追加」をタップ
5. iPad のホーム画面に **アリのアイコン** が追加されます
6. アイコンをタップ → **全画面で起動** (Safari の URL バーが消える)

### アプリ化後にできるようになること

- ✅ **オフライン動作** — 初回アクセス後は Wi-Fi なしでも起動できる (Service Worker が HTML/JS をキャッシュ)
- ✅ **専用アイコン** — ホーム画面で他のアプリと並んで表示
- ✅ **全画面起動** — Safari の UI が見えない
- ✅ **状態の独立** — ブラウザの履歴やタブとは別管理
- ✅ **更新通知** — 新しいバージョンを公開すると、起動時に下部に通知バナーが出る

### 家族で共有する場合

家族の各 iPad でも同じ手順を繰り返すだけです。各端末はそれぞれが Service Worker でローカルキャッシュを持ちます。
GitHub Pages 側を更新すれば、各端末が次回起動時に検知して「新しいバージョンがあります」と表示されます。

### サイトを更新したとき

新しいバージョンを GitHub Pages にデプロイした後、各 iPad でアプリを起動すると下部に
**「新しいバージョンがあります [更新] [あとで]」** のバナーが出ます。
「更新」をタップすると最新版に切り替わります（自動リロード）。

## 動作要件

- **iPadOS 14 以上** (iPadOS 14.5+ 推奨: PS5 ゲームパッドが使える)
- Safari (またはその他の Web 標準準拠ブラウザ)
- M1/M2/A シリーズ全般で 60fps 動作確認

## デスクトップ版との違い

| 項目 | デスクトップ版 (`../`) | iPad 版 (このフォルダ) |
|---|---|---|
| デバッガパネル | あり | なし (画面を最大化) |
| 入力 | キーボード + ゲームパッド | + タッチパッド |
| レイアウト | 固定 (1024+380) | レスポンシブ |
| PWA 対応 | なし | あり |
| ROM 読み込み | D&D + ファイル選択 | ファイル選択のみ |

エミュレーターコア (`src/*.js`) はデスクトップ版と同一で、両方で同じ修正を共有してきました。
